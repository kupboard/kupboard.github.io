Collection

collection은 kupboar cluster에 설치할 software패키지들과, 예를 들어 nginx 또는 redis 유저가 배포하고 싶은 (contianer로 배포가 가능한) 서비스가 포함된 하나의 묶음이다.



collection은 package와 application 그리고 mugset.yaml으로 구성되며 kupboard는 default collection을 포함하고 있으며 지정된 action을 통해서 클러스터에 배포할 수 있다. default collection 정보는 아래 섹션에서 확인이 가능하다. Deault Package



유저가 자신만의 collection을 구성하면 kupboard를 통해서 배포할 수 있다. 이는 프로젝트 별로 또는 서비스별로 구분해서 관리가 필요할 경우 유용하다.

collection에 포함된 패지키는 일반적인 오프소스 소루션이 될 수 있으며 install or uninstall등을 위한 간단한 ansible script 있으면 된다. 서비스 는 Dockerfile이 포함된 with or without 소스 디렉토리만 있으면 된다.



하나의 collection 포함되는 mugset.yaml은 collection에 포함된 패지키와 서비스의 정보가 정의되어 있다. 유저는 각 패키지와 서비스 배포시 필요한 action을 정의할 수 있으며 각 action에서 요구되는 parameter도 지정할 수 있다. mugset는 kupboard가 자동으로 로딩하고, 유저가 정의한 action을 통해서 원하는 패키지와 서비스를 배포할 수 있다. Collection은 중복되지 않은 이름으로 디렉토리로 data/collections에 위치해야 한다.

tip

collection은 "colleciton-name.tar.gz"과 같이 archive파일로 생성하여 추가하면 자동으로 압축해제된다.

How to deploy collection?#

Package#

Deployment of a default package

$ kupboard collection package -n <package-name> [-a <action-name>]

Deployment of all default packages

$ kupboard collection package

Deployment of a customer package

$ kupboard collection package -c <collection-name> -n <package-name> [-a <action-name>]

Application#

Build customer application

$ kupboard collection app -c <collection-name> -n <app-name>

Build all customer applications

$ kupboard collection app -c <collection-name>

Build all customer applications from all customer collections

$ kupboard collection app
