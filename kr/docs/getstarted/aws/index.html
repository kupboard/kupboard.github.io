AWS Marketplace

Introduction#

kupboard contianer는 Docker Container가 실행될 수 있는 어떤 환경에서든 실행이 가능하다. 이 섹션에서는 AWS Marketplace에 등록된 kupboard container를 설치 및 실행방법을 설명하고, kupboard 명령어를 사용하여 간단한 구성의 Cloud Native 환경울 구축하는 방법을 설명한다.

note

If you use a kupboard container of Dockerhub instead of AWS Marketplace, skip the subscription process and move to Servers to create EC2 instances.

Subscription#

Kupboard를 사용하기 위해서는 먼저 AWS Marketplace에서 kupboard를 검색한 후 [Continue to Subscribe]를 선택하여 구독한다.



구독과정이 완료되면 [Continue To Configuration]을 선택하여 설정화면으로 이동한다.



설정 메뉴에서 Delivery Method와 Software Version을 선택 후 [Continue to Launch]를 선택한다.



이제 Kupboard를 실행할 준비가 완료되었다. 하단의 [View container image details] 링크를 선택하면 Kupboard 컨테이너 이미지를 Pull하기 위한 명령어를 확인할 수 있다.



Installation#

AWS Marketplace에 등록된 container를 사용하기위해서는 먼저 awscli를 설치해야 한다. 만약 설치되지 않았다면 Installing, updating, and uninstalling the AWS CLI를 방문하여 설치한다.

note

awscli 명령어를 사용하기 전에 AWS 계정의 credentail을 아래와 같이 정의해야 한다. 만약 awscli에 익숙하지 않은 사용자라면 Set up AWS Credentials and Region for Development를 참고하길 바란다.

export AWS_ACCESS_KEY_ID=
export AWS_SECRET_ACCESS_KEY=

Login to registry#

AWS Marketplace에서 구독한 kupboard container를 사용하기 위해서는 먼저 AWS Marketplace의 registry에 로그인해야 한다.

$ aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 709825985650.dkr.ecr.us-east-1.amazonaws.com

Pull kupboard image#

$ docker pull 709825985650.dkr.ecr.us-east-1.amazonaws.com/atop-cloud/kupboard:0.9.0

Run kupboard#

kupboard container가 정상적으로 pull되었다면 docker run 명령어를 사용하여 아래와 같은 결과가 출력될 것이다.

Bash script for running kupboard (Recommended)#

kupboard container를 실행하기 위해 docker 명령어를 직접사용하는 대신 아래와 같은 방법으로 bash script를 생성해서 사용하면 훨씬 간단하게 사용할 수 있다.

#!/bin/bash
docker run --rm -it 709825985650.dkr.ecr.us-east-1.amazonaws.com/atop-cloud/kupboard:0.9.0 "$@"

bash script를 사용하여 kupboard 명령어를 실행하는 예제이다.

$ kupboard setup --init-user

data 폴더를 마운트 할 경우에는 아래와 같이 -v 옵션을 추가한다.

#!/bin/bash
docker run --rm -it -v $(pwd)/data:/kupboard/data 709825985650.dkr.ecr.us-east-1.amazonaws.com/atop-cloud/kupboard:0.9.0 "$@"

Servers#

Kupboard의 클러스터는 서비스와 용도에 따라서 다양한게 구성될 수 있다. 여기에서는 1개의 서버로 구성된 Admin 클러스터, 1개의 서버로 구성된 Gateway클러스터 그리고 4개의 서버로 구성된 Service클러스터를 생성하여 전체 시스템을 구성할 예정이다.

note

kupboard의 0.9 버전에서는 EC2 인스턴스의 생성 및 관리를 수동으로 진행하지만, 이후 버전에서는 awscli를 통하여 인스턴스까지 자동으로 관리하는 기능이 추가될 예정이다.

EC2 Instance#

아래 테이블과 같이 6개의 인스턴스를 생성한다. 모든 인스턴스의 OS는 Ubuntu18.04 또는 20.04 이어야 하며 모든 인스턴스는 동일한 key pair를 사용해야 한다.

ClusterEC2 TypeQuantityBoot VolumeEBS Volume
Admint3.meidum <150GBN/A
Gatewayt3.meidum <130GBN/A
Servicet3.large <430GB50GB
info

Service클러스터를 구성하는 인스턴스들은 EBS Volume이 추가되어야 한다. 이는 kupboard가 클러스터를 초기화하는 과정에서 자동으로 컨트롤되기 때문에 별도의 포맷이나 마운트 작업을 필요로 하지 않는다.

note

kupboard-0.9에서는 Ubuntu가 설치된 Host만 지원하지만 이후 버전에는 Centos가 설치된 Host도 지원될 예정이다.
