Get Started

이 섹션에서는 Kupboard를 실행하는 방법과 Configuration에 대해서 간략히 살펴본다.

Installation#

Kupboard는 도커 컨테이너 형태로 제공되며 아래와 같이 간단한 명령어로 실행이 가능하다.

$ docker run kupboard.io/kupboard

아래는 -h or --help 옵션을 입력한 결과이다.

$ docker run kupboard.io/kupboard --help
kupboard is a root command for managements of the Kupboard platform.
Usage:
kupboard [flags]
kupboard [command]
Available Commands:
build Build application containers
collection Deploy applications and install packages
deploy Deploy services on target clusters
help Help about any command
setup Initialize target clusters
ssh ssh connection to a target host
Flags:
--config string config file (default "kupboard.yaml")
--debug enable debug
-e, --envvar string ansible envvars
-h, --help help for kupboard
--hide-log diable logging
--hide-logo hide kupboard logo
Use "kupboard [command] --help" for more information about a command.

Data#

Kupboard를 사용하여 서비스환경을 구축하기 위해서는 컨테이너 실행시 data 폴더를 마운트시켜야 한다. data 폴더의 구조는 아래와 같다.

data
├── api
├── certs
├── collections
├── log
├── manifests
└── kupboard.yaml
$ docker run -v $(pwd)/data:/kupboard/data kupboard.io/kupboard

Configuration#

Kupboard가 기본으로 사용하는 설정파일의 경로는 data/kupboard.yaml이다. 사용자가 설정파일의 경로를 지정하지 않는다면 이 기본파일이 적용될 것이다. 만약 다른 이름의 설정파일을 사용하거나, 하나 이상의 설정파일을 사용하다면 --config 옵션으로 파일을 지정해야 한다.

data
├── kupboard.yaml
└── myconfig.yaml
$ docker run -v $(pwd)/data:/kupboard/data kupboard.io/kupboard --config data/myconfig.yaml
